⭒ Released in 2010.

⭒ JavaScript library that includes classes and functions.

⭒ MVC Type = Model + View + Controller

⭒ The user interacts with the system through the controllers

⭒ The controller perceives the user’s actions, interprets them, and informs the model of changes required in accordance with the actions

⭒ The model is in charge of providing access to the data available in the system. Receives a change request from the controller, executes it.

⭒ The processes are invisible to the user – their task is to press a button and get the result (blackbox).

⭒ Main task: organize the client part of the application in a logical way and to synchronize changes with the server part.


⭒ Backbone.js uses:

- Models

- Views

- Collections

- Events

- Routers



❅ Models: all data are presented on the client side in this form. Can be created, deleted, validated, and stored on the server. These are triggered through changing the model attribute in the interface.

❅ Views: receives information about the model (data) change, and changes accordingly. Each view can be updated independently of each other since it is based on a specific model. These views collectively form the interface of the application.

❅ Collections: group Models together (like a JavaScript array)

❅ Events: bind events like user clicks or mouse-overs to the code

❅ Routers: provides routing methods on the client side – in charge for the way the application will change with altering URL in the browser.


❅ Popularity:
Backbone has started to decline in popularity as ReactJS and AngularJS is preferred by developers. It is considered to be useful for only "legacy" code.




❅ Strengths:

⭒ Easy to understand usability modules, straightforward learning curve

⭒ Frameworks and libraries are designed to accelerate the development process through ready-made templates and solutions and elaborated structure

⭒ Mostly open source code that can be used for free – total cost of developing applications decreases

⭒ Well documented

⭒ Lightweight set of tools – alone it is 7.6KB gzipped, with Jquery and Underscore a total of 41.6KB.

⭒ Very compact library – simplest minimalist set of classes and functions

⭒ Store data in models instead of in DOM

⭒ Aims to serve as foundation for the application, provides options for extending usage of other libraries and tools (hence the name Backbone)

⭒ Flexible and combines well with other tools – only fixed dependency is Underscore.js library as it does not work without it

⭒ Event-driven communication between views and models

⭒ Syncing with a back-end

⭒ Maintainable by following conventions

⭒ Simplifies complex UI design – saves time, increases code readability, simplifies future recoding


❅ Weaknesses:

⭒ May be challenging for beginners

⭒ Views become heavy as the application extends

⭒ No built-in support for data bindings, must rely on add-ons

⭒ The more complex the app, the more third-party plugins and custom coded script is needed to make the app work

⭒ Difficulties with testing code

⭒ Slow release of updates

⭒ Lack of features


❅ Who is it used by?

⭒ Most popularly used for eCommerce websites because it is light and helps prevent unnecessary requests to the server. Following are Business, News, and Sports websites.
